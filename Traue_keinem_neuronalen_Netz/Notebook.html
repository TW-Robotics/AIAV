Trauen Sie keinem neuronalem Netz¶

In diesem Notebook finden Sie Python Programmcode um basierend auf dem LRP und GradCAM Gründe für CNN Entscheidungen zu visualisieren. Bevor Sie beginnen und dieses Notebook verwenden können müssen Sie folgende Rahmenbedingungen installieren:

  • Python2 oder Python3: Achtung: gegebenenfalls müssen Sie Programmcode auf Ihre Version anpassen
  • Folgende Python Packages müssen Sie installieren:
    • tensorflow oder tensorflow-gpu: je nachdem ob Sie eine NVIDIA Grafikkarte und deep learning Treiber installiert haben
    • numpy: Numpy benötigen Sie für grundlegende Matrixoperationen
    • keras: Das erleichtert und die Arbeit mit tensorflow. Achtung: Sollten Sie mit tensorflow Version 2 oder höhere Arbeiten brauchen Sie Keras nicht separat installieren
    • matplotlib: Für Visualisierungen
    • keras-innvestigate: für LRP oder andere CNN Visualisierungen

Weiters benötigen wir Daten in Bildform. In diesem Beitrag verwenden wir einen sehr einfachen Datensatz: den fashion-mnist Datensatz. Die Rohversion dieses Datensatzes können Sie sich hier downloaden. Eine Vorverarbeitete Version ist finden Sie in unserem Repository im Unterordner zu diesem Tutorial.

In der README Datei zu diesem Ordner finden Sie Installationsanleitungen.

Worum gehts?¶

Sie haben bereits Convolutional Neuronal Networks (CNNs) kennengelernt (z.B.: hier auf unserer Plattform). Bisher haben wir CNNs als Framework für Klassifizierung von Bilder und Lokalisierung von Objekten kennengelernt.

Wir beschäftigen uns in diesem Tutorial mit der Frage Was hat das CNN tatsächlich gelernt?. Es muss Gründe geben, warum der Output des CNN eine bestimmte Klasse für wahrscheinlicher hält. Im Vergleich zu CNN Anwendungen und Weiterentwicklung existiert hier relativ wenig Forschung. Eine Technik zur Visualisierung der Entscheidungsgründe ist LRP oder GradCAM. Mit diesen Techniken werden wir uns in diesem Tutorial beschäftigen.

Ein weitaus schlimmeres grundsätzliches und häufig nicht sichtbares Problem basiert auf den Daten, die zum Training des CNNs verwendet werden. Vereinfacht kann man dieses Phänomen mit folgendem Beispiel erläutern: Wir wollen ein CNN auf einem Datensatz trainieren und Pferde detektieren! Also nutzen wir bestehende Datenbanken dazu. Was wir nicht wissen ist, dass im Datensatz ein systematischer Fehler enthalten ist: Jedes Pferdefoto hat in linken unteren Eck ein Wasserzeichen. Lernt das CNN nun Pferde zu klassifizieren oder Wasserzeichen? Wie kann man das herausfinden? Dieses Problem ist intensiv in dieser Publikation diskutiert.

In diesem Tutorial geht es um ähnliche Dinge: wir wollen unterschiedliche Kleidungsstücke klassifizieren und anschließend feststellen ob die Gründe dieser Klassifizierung valide sind! Dazu implementieren wir folgendes Vorgehen:

  1. Datensatz erzeugen
  2. CNN trainieren
  3. CNN Performance analysieren
  4. LRP und GradCAM zur Visualisierung nutzen

Das weitere Vorgehen dieses Tutorials diskutiert dieses Vorgehen.

Der Datensatz¶

Für dieses Tutorial wurden der fashion-mnist Datensatz (siehe oben) verwendet. Wir ziehen 10000 Beispielbilder aus dem Trainingsdatensatz und evaluieren mit dem Testdatensatz. Folgende Ordnerstruktur ist vorhanden:

  1. TRAIN Ordner: hier sind Beispiele der Bildklassen (z.B.: ein Bild eines T-Shirts) abgelegt. Für jede der neun Bildklassen ist ein Subfolder angelegt. Diese Struktur ist verpflichtend für das verwendete Softwareframework.
  2. TEST Ordner: die gleiche Struktur wie im TRAIN Ordner - allerdings dienen diese Bilder nur zum Testen des CNNs.

CNN fine-tuning¶

Wir nutzen das VGG-16 CNN vortrainiert auf imagenet. Dieses vortrainierte Modell (Achtung: das automatische Herunterladen des vortrainierten Models kann dauern) werden wir dann auf unsere Daten anpassen.

In [2]:
vgg_conv = VGG16(weights='imagenet', #Wir laden VGG16 und nutzen die vortrainierten Gewichtungen
                 include_top=False,  #Wir wollen das Netzwerk an unsere Daten anpassen -> das 'top' benötigen wir nicht
                 input_shape=(image_size, image_size, 3) #Alle Bilder werden auf 224x224x3 Pixel skaliert
                )
for layer in vgg_conv.layers[:-4]: #Diese Layer sollen nicht weiter trainiert werden
    layer.trainable = False #Training deaktivieren
#--- Jetzt bauen wir uns das Netzwerk für unsere Zwecke um ---#
last = vgg_conv.output #Wir nehmen den abgeschnittenen Output des VGG16
x = Flatten()(last)                 #... und machen einen Vektor aus dem output
x= Dense(1024,activation='relu')(x) #... und setzen einen fully connected Layer darauf
x=Dropout(0.5)(x)                   #... und fügen einen Dropout Layer dazu
pred=Dense(10,activation='softmax')(x) #... und definieren einen neuen Output Layer
#Anmerkung: 10 Neuronen am Output weil wir 10 Klassen haben
model = Model(vgg_conv.input,pred)  #Wir 'kleben' das Modell zusammen
#---  Show a summary of the model ---#

model_wo_softmax = model #Model in eine neue Variable schreiben
LPR_10 = innvestigate.analyzer.relevance_based.relevance_analyzer.LRPAlphaBeta(model_wo_softmax,1,0) #LRP implementierung
ground_truth = test_data[1] #In dieser Variable stehen die wahren Klassenlabels 
labels_ground_truth = np.argmax(ground_truth,1) #Berechnung eines target Vektor - nun sind die Eitnräge die Klassen
for class_ID in range(0,10):
    test_index = np.where(labels_ground_truth==class_ID)[0][0] #Das ist der Index des ersten Testbildes mit ausgewähltem Index
    #--- gradCAM Berechnung ---#
    grads = visualize_cam(model, layer_idx, filter_indices=class_ID, seed_input=test_data[0][test_index], backprop_modifier='guided')
    #--- LRP Berechnung ---#
    LRP = LPR_10.analyze(test_data[0][test_index].reshape((1,224,224,3)))
    #--- Prüfen ob das Objekt richtig erkannt wurde ---#
    if(ground_truth[test_index,class_ID] != 1):
        postfix = " (falsch erkannt)"
    else:
        postfix = ""
    #--- Visualisierung der Daten ---#
    fig, (ax1, ax2, ax3) = plt.subplots(1, 3)
    fig.suptitle("Visualisierung der Klasse "+str(class_ID) + postfix)
    ax1.imshow(test_data[0][test_index]); ax1.axis('off')
    ax2.imshow(grads); ax2.axis('off')
    ax3.imshow(LRP[0,:,:,0]); ax3.axis('off')

Die Abbildungen zeigen immer das Rohbild (links), die gradCAM Visualisierung (mittig) und die LRP Visualisierung (rechts). Sehen wir uns die Klassen nacheinander an:

Der Gesamteindruck zeigt allerdings, dass die nicht alle Entscheidungen des CNN nachvollziehbar sind.

Wir haben hier mit einem sehr simplem Datensatz gearbeitet - bei komplexeren Datensätzen oder Datensätzen mit wesentlich weniger Beispielen kann diese Visualisierungstechnik wertvollen Einblick in das Verhalten eines CNNs geben.
